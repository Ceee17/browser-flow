# Product Vision

BrowserFlow is an **AI-Native Browser Automation Platform** built for the cloud era.

## Mission

Replace brittle, desktop-first legacy RPA with a platform that is:

- **Browser-first** — web apps are the primary interface for modern business.
- **JSON-native** — workflows are data, not opaque binaries.
- **Cloud-native** — control plane is SaaS; execution agents are outbound-only.
- **Open core** — the execution stack is community-driven; the control plane is commercial.

## Target Market

- Engineering teams doing QA, scraping, E2E testing, and ops automation.
- Organizations that need browser automation without desktop-binding agents.
- Developers who want version-controlled, API-accessible automation instead of point-and-click macro recorders.

## Long-term Vision

Evolve from browser automation into a **Universal Automation Platform**:

1. Browser Automation (v1)
2. Desktop Automation (v2)
3. API Automation (v2)
4. Database Automation (v2)
5. SSH Automation (v2)
6. Mobile Automation (v3)
7. AI Agents + MCP (v2+)

The architecture must be modular so each new domain is additive.

## Differentiation

| Legacy RPA | BrowserFlow |
|------------|-------------|
| Desktop automation first | Browser-first (web-native) |
| Proprietary workflow formats | Open JSON schema |
| Centralized automation server | Distributed, outbound-only agents |
| Selenium / legacy drivers | Playwright / CDP |
| XPath-only selectors | Multi-strategy + confidence scoring |
| Closed-source core | Apache 2.0 core |
| No AI self-healing | AI-assisted, optional |
| Heavy Windows footprint | Cross-platform, Docker-first |
